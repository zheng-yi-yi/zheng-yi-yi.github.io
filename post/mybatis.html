MyBatis 快速入门

发表于2023-02-26|更新于2023-02-28|JavaWeb
|阅读量:

今天我们来做一个简单的案例，来实现 mybatis 的快速入门，

简单来说，该案例的需求就是查询 user 表中的所有数据，把 user 表中的所有数据查出来，封装到 User 类中，再将一个个 User 实例装到 List 集合里头。

下面是完成这个 mybatis 快速入门案例的相关步骤。

一、创建表

首先我们需要创建一个 User 表并添加若干数据，这一步操作只需执行以下 SQL 语句即可。

create database mybatis;

use mybatis;

drop table if exists tb_user;

create table tb_user(
	id int primary key auto_increment,
    username varchar(20),
    password varchar(20),
    gender char(1),
    addr varchar(30)
);

INSERT INTO
	tb_user
VALUES
(1, '张三', '123', '男', '北京');

INSERT INTO
	tb_user
VALUES
(2, '李四', '456', '女', '上海');

INSERT INTO
	tb_user
VALUES
(3, '王五', '789', '男', '广州');

二、导入坐标

现在，我们使用 Maven 来构建项目，下面创建一个模块，如图所示：

创建完模块后，我们需要在 pom.xml 中导入 MyBatis 的相关依赖坐标，才能够使用 MyBatis ，实例如下（其中 x.x.x 是版本号）：

<dependency>
  <groupId>org.mybatis</groupId>
  <artifactId>mybatis</artifactId>
  <version>x.x.x</version>
</dependency>

除了导入 MyBatis 依赖，我们还需要导入其他依赖，比如数据库连接，单元测试等。这里给出今天快速入门案例中的pom.xml 文件，如下所示：

<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <groupId>org.example</groupId>
    <artifactId>myBatis-demo</artifactId>
    <version>1.0-SNAPSHOT</version>

    <dependencies>
        <!-- mybatis 依赖 -->
        <dependency>
            <groupId>org.mybatis</groupId>
            <artifactId>mybatis</artifactId>
            <version>3.5.5</version>
        </dependency>

        <!-- mysql 驱动 -->
        <dependency>
            <groupId>mysql</groupId>
            <artifactId>mysql-connector-java</artifactId>
            <version>5.1.46</version>
        </dependency>

        <!-- junit 单元测试 -->
        <dependency>
            <groupId>junit</groupId>
            <artifactId>junit</artifactId>
            <version>4.13</version>
            <scope>test</scope>
        </dependency>
    </dependencies>

</project>

导入依赖后记得点击 Maven 刷新按钮。

三、MyBatis 核心配置文件

接下来就到了 MyBatis 核心配置文件的编写工作了。

需要注意， MyBatis 核心配置文件主要用于配置数据库的连接信息，该连接信息在之前的 JDBC 编程中是写在 Java 代码中的，现在我们把这些连接信息写在 XML 配置文件中，这也解决了硬编码的问题。

备注，XML 配置文件中包含了对 MyBatis 系统的核心设置，包括：

  • 获取数据库连接实例的数据源（DataSource）
  • 决定事务作用域和控制方式的事务管理器（TransactionManager）

那这个核心配置文件该怎么写？举一个例子：

这里我们创建一个文件 mybatis-config.xml ，用于编写 MyBatis 核心配置文件。那么这个mybatis-config.xml 文件中需要编写什么样的内容？

下面给出一个简单的示例：

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE configuration
        PUBLIC "-//mybatis.org//DTD Config 3.0//EN"
        "https://mybatis.org/dtd/mybatis-3-config.dtd">
<configuration>
    <environments default="development">
        <environment id="development">
            <transactionManager type="JDBC"/>
            <dataSource type="POOLED">
                <!-- 数据库的连接信息 -->
                <property name="driver" value="com.mysql.jdbc.Driver"/>
                <property name="url" value="jdbc:mysql:///mybatis?useSSL=false"/>
                <property name="username" value="root"/>
                <property name="password" value="1234"/>
            </dataSource>
        </environment>
    </environments>
    <mappers>
        <!--加载 SQL 映射文件-->
        <mapper resource="UserMapper.xml"/>
    </mappers>
</configuration>

上方代码罗列出了 XML 核心配置文件的关键部分：

  • 头部声明：用于验证 XML 文档的正确性
  • environment元素体：包含了连接池和事务管理的配置。
  • mappers 元素：包含了一组映射器（mapper），用于加载对应的 SQL 映射文件。

四、创建 SQL 映射文件

当我们配置完 MyBatis 核心配置文件之后，就可以来编写 SQL 映射文件。步骤如下：

SQL 映射文件就是将 SQL 语句编写到 XML 配置文件中去，在 MyBatis 中，一个 SQL 语句可以通过 XML 定义，这可以高效统一地管理 SQL 语句，也能解决硬编码的问题。在一个 SQL 映射文件中，可以定义无数个映射语句。

事实上 MyBatis 提供的所有特性都可以利用基于 XML 的映射语言来实现。

下面是 UserMapper.xml的代码示例：

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper
        PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "https://mybatis.org/dtd/mybatis-3-mapper.dtd">

<!-- namespace :  名称空间 -->
<mapper namespace="test">
    <select id="selectAll" resultType="com.zcst.pojo.User">
        select * from tb_user;
    </select>
</mapper>

在上方代码中：

  • 我们在命名空间 "test" 中定义了一个名为 selectAll 的映射语句
  • 其中namespace称为命名空间，而id是这条语句的唯一标识， resultType 是对应返回结果的类型（将来查询后返回的数据要包装成什么样的类型就写什么类型）。
  • 将 SQL 语句编写成 映射文件的好处是，在之后的 Java 代码中，我们可以通过全限定名 "test.selectAll" 来调用这条映射语句。这种方式与调用 Java 对象的方法类似。
  • 就这样，通过名称空间 namespace 以及语句名称 id ，我们可以将这条 select 语句匹配到对应名称、参数和返回类型的方法，在对应的映射器接口中调用方法。

另外，除了使用 XML 来完成 SQL 语句映射，我们还可以使用 Java 注解来配置语句映射，这种方式对于一些简单的SQL语句来说，会使得最后的 Java更加简洁。但是对于复杂一点的 SQL 语句，利用Java 注解来完成语句映射的工作就不如 XML。

五、编码

当我们将这些配置文件编写完成之后，就可以开始编码部分。

1、User类

首先是实体类的创建，这里我们定义了一个 User 类，这个类和我们要查询的 tb_user 表是对应的。

如下所示：

package com.zcst.pojo;

public class User {
    private Integer id;
    private String username;
    private String password;
    private String gender;
    private String addr;

    public Integer getId() {
        return id;
    }

    public void setId(Integer id) {
        this.id = id;
    }

    public String getUsername() {
        return username;
    }

    public void setUsername(String username) {
        this.username = username;
    }

    public String getPassword() {
        return password;
    }

    public void setPassword(String password) {
        this.password = password;
    }

    public String getGender() {
        return gender;
    }

    public void setGender(String gender) {
        this.gender = gender;
    }

    public String getAddr() {
        return addr;
    }

    public void setAddr(String addr) {
        this.addr = addr;
    }

    @Override
    public String toString() {
        return "User{" +
                "id=" + id +
                ", username='" + username + '\'' +
                ", password='" + password + '\'' +
                ", gender='" + gender + '\'' +
                ", addr='" + addr + '\'' +
                '}';
    }
}

2、创建测试类

最后，我们创建一个测试类 MyBatisDemo，在这个测试类中，我们可以尝试编写一些关键代码来使用 MyBatis。

（1）获取 SqlSessionFactory

代码如下：

String resource = "mybatis-config.xml";
InputStream  inputStream = Resources.getResourceAsStream(resource);
SqlSessionFactory sqlSessionFactory = new SqlSessionFactoryBuilder().build(inputStream);

在上述代码中：

  • 首先我们定义了一个字符串，并保存配置文件 mybatis-config.xml 的相对路径；

  • 接着调用Resources对象的 getResourceAsStream方法来获取一个 InputStream （字节输入流）实例

  • 最后调用 SqlSessionFactoryBuilder 的 build 方法来获取一个 SqlSessionFactory 的实例。

备注：

  • 每个基于 MyBatis 的应用都是以一个 SqlSessionFactory 的实例为核心的。
  • Resources是MyBatis中的一个工具类，它包含一些实用的方法，方便我们加载资源文件。

（2）获取 SqlSession 对象

代码如下：

SqlSession sqlSession = sqlSessionFactory.openSession();

上面代码中：

  • 通过调用 sqlSessionFactory 对象的 openSession 方法，我们可以获取一个 SqlSession 对象。
  • 有了这个 SqlSession 对象之后，我们就可以用它来执行 SQL 语句。

（3）执行 SQL 语句

代码如下：

List<User> user = sqlSession.selectList("test.selectAll");

在上述代码中：

  • 由于今天这个快速入门的案例是查询 tb_user 表中的所有数据；
  • 那么现在我们就可以调用 sqlSession 对象的 selectList 方法；
  • 在这个selectList 方法中，我们需要传入参数，这个参数就是 SQL 语句的唯一标识，即test.selectAll （也就是以 namespace.id 的形式来构成 SQL 语句的唯一标识）
  • 由于方法返回的是一个 List 集合，因此我们将查询的结果封装到 List<User>中。

、

（4）打印测试

代码如下：

for(User u : user) {
    System.out.println(u);
}

（5）释放资源

代码如下：

sqlSession.close();

（6）完整代码

MyBatisDemo 测试类的完整代码如下：

package com.zcst;

import com.zcst.pojo.User;
import org.apache.ibatis.io.Resources;
import org.apache.ibatis.session.SqlSession;
import org.apache.ibatis.session.SqlSessionFactory;
import org.apache.ibatis.session.SqlSessionFactoryBuilder;
import java.io.IOException;
import java.io.InputStream;
import java.util.List;

/**
 *  Mybatis 快速入门代码
 */
public class MyBatisDemo {
    public static void main(String[] args) throws IOException {

        // 1. 加载 mybatis 的核心配置文件，获取 SqlSessionFactory
        String resource = "mybatis-config.xml";
        InputStream  inputStream = Resources.getResourceAsStream(resource);
        SqlSessionFactory sqlSessionFactory = new SqlSessionFactoryBuilder().build(inputStream);

        // 2. 获取 SqlSession 对象
        SqlSession sqlSession = sqlSessionFactory.openSession();

        // 3. 执行 SQL 语句，并封装结果集
        List<User> user = sqlSession.selectList("test.selectAll");

        // 4. 打印封装好的结果集
        for(User u : user) {
            System.out.println(u);
        }

        // 5. 释放资源
        sqlSession.close();

    }
}
